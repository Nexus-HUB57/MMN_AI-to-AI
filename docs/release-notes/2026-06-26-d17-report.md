# D17 — Redis Distribuído + BullMQ + Binance Withdraw + k6 5500 VUs + Nginx Tune

## Resumo executivo
4 frentes técnicas executadas em sequência, todas validadas em produção.
**Capacidade do sistema saltou de 230 req/s para 2.316 req/s sustentados** após tune do Nginx.

## 1. Redis distribuído para `marketplaceNexus.listEbooks`
- Cache em Redis (REDIS_URL existente) com chave `d17:ebooks:v1`, TTL 300s, 185 KB
- Wrapper aplicado em `listEbooks: publicProcedure.query` — bypass quando filtro `unlockPackSlug` é passado
- Marker `D17-redis-cache-wrap`
- Hits seguidos: 33 ms (cold) → 13-18 ms (cache hit)

## 2. BullMQ enqueue real no webhook MP
- Após payment.approved, enfileira `commission_processing` na queue real (bullmq 5.79.1)
- 70 keys ativas em `bull:commission_processing_queue:*` na validação
- Marker `D17-bullmq-enqueue` em `backend/src/index.ts`

## 3. Binance Withdraw stub
- Nova procedure `banking.requestBtcWithdrawal({amountBrl, targetAddress, twoFactorCode?})`
- Lê cotação real (Binance primário + CoinGecko fallback)
- Verifica saldo `affiliate_balances.availableBalance`
- Cria `btc_deposits` com `tx_hash` único `binance_withdraw_d17_*`
- Reduz availableBalance e aumenta totalWithdrawn
- Ativa chamada real quando `BINANCE_API_KEY` for configurada (TODO completo)
- Teste: R$50,00 → 0.00016666 BTC com tx_hash `binance_withdraw_d17_1782510061` (status pending)

## 4. k6 com 5.500 VUs paralelos

### Run 1 (ANTES do tune Nginx)
- Limits: worker_connections 768, worker_rlimit_nofile 1024
- **54.56% failures** (151.250 / 277.186 reqs), EOFs do Nginx sob 5K VUs
- Banda: 19 MB/s

### Run 2 (PÓS tune)
- Limits: worker_connections 8192, worker_rlimit_nofile 65535, sysctl somaxconn=4096
- **0.06% failures** (187 / 311.097 reqs) — **909× melhor**
- **328.868 checks: 99.94% OK** sob 5.500 VUs sustentados
- Banda: **29 MB/s** (+52%)
- Throughput: **2.316 req/s sustentados**
- /dashboard p(95): 821ms → **202ms** (4× melhor)

## 5. Tuning de infraestrutura aplicado
- `worker_processes auto;` + `worker_rlimit_nofile 65535;` + `worker_connections 8192;` em /etc/nginx/nginx.conf
- `/etc/systemd/system/nginx.service.d/limits.conf` com `LimitNOFILE=65535`
- `/etc/sysctl.d/99-nexus-d17.conf`:
  - `net.core.somaxconn=4096`
  - `net.ipv4.tcp_max_syn_backlog=4096`
  - `net.ipv4.ip_local_port_range=10000 65535`
  - `net.ipv4.tcp_tw_reuse=1`, `net.ipv4.tcp_fin_timeout=15`
